🤙 Homerow Mods — Hold for Modifiers

Each home row key does two things: tap = letter, hold = modifier. Cross-hand only — hold a left-hand key while pressing a right-hand key (or vice versa). Same-hand presses always type letters.

Hold time is 280ms. Type normally and they never misfire. If you need the same-hand modifier (e.g. Ctrl+T both left hand) use the sticky mods on NAV layer instead.

📋 Same-Hand Problem — When Ctrl and Target Are Both Left

Homerow mods require opposite hands. If the modifier key and the letter you want are on the same hand, use sticky mods from NAV instead:

  1. Hold Space to enter NAV layer
  2. Tap the sticky mod (A=GUI · R=Alt · S=Shift · T=Ctrl) — you can stack multiples
  3. Release Space
  4. Press your target key
Example: Ctrl+F (find) — both Ctrl (T) and F are left hand. Solution: Space → tap T (sticky Ctrl) → release Space → tap F.
📱 Using with iPad

The Corne connects to iPad over Bluetooth LE. iPadOS treats it as a standard hardware keyboard. Most things work exactly as on a PC once you know the key mapping.

Pairing

  1. Put the keyboard into Bluetooth pairing mode — hold FN+NUM (both layers) to enter SYS, then tap BT0–BT3 to select a profile slot. If it's a fresh slot it will start advertising.
  2. On iPad: Settings → Bluetooth → wait for "Corne Choc Pro BT" to appear under Other Devices.
  3. Tap it to pair. It may ask for a PIN — type it on the keyboard and press Enter.
  4. To switch between devices (e.g. PC and iPad) use SYS layer: FN+NUM, then tap BT0/BT1 etc.
Each half of the split keyboard connects independently. Make sure both halves are connected — check Settings → Bluetooth for both entries.

Alt+Tab on iPad

The ZMK swapper (Space+W) holds Alt and sends Tab. On iPad this sends ⌥+Tab which is not the standard app switcher (that's ⌘+Tab). For app switching on iPad use A (GUI) held + R+S Tab combo instead.

Notes

iPadOS shows a keyboard shortcut overlay in most apps when you hold ⌘ — hold your A homerow mod for a second to see what's available in the current app.
The combos (X+C for copy, C+D for paste) use Ctrl+Insert / Shift+Insert which are PC shortcuts — they do NOT work on iPad. Use ⌘+C / ⌘+V via the GUI homerow mod instead.

🐧 Using with Linux

ZMK sends standard HID codes that Linux understands natively. Pairing over Bluetooth, and most shortcuts work the same as on Windows with one main difference: GUI = Super key.

Bluetooth Pairing (command line)

  1. Open a terminal and run: bluetoothctl
  2. Type: power on
  3. Type: scan on
  4. Put keyboard into pairing mode (SYS layer → BT slot)
  5. When "Corne Choc Pro BT" appears, type: pair XX:XX:XX:XX:XX:XX (use the MAC shown)
  6. Type: connect XX:XX:XX:XX:XX:XX
  7. Type: trust XX:XX:XX:XX:XX:XX (auto-reconnects on boot)
  8. Type: scan off then exit
GNOME users: Settings → Bluetooth is easier. KDE users: System Settings → Bluetooth. Both work fine.

Terminal Copy/Paste — Important Difference

In Linux terminals (bash, zsh, etc.), Ctrl+C kills the running process — it does not copy. Copy/paste in terminals uses Ctrl+Shift+C and Ctrl+Shift+V.

Ctrl+Alt+Delete on Linux

On most Linux desktop environments Ctrl+Alt+Delete shows a logout/power dialog (like Windows). Use the same method:

Hold Space→tap R(sk Alt)→tap T(sk Ctrl)→tap Y(Del on NAV)

In TTY (no GUI) Ctrl+Alt+Del may reboot the machine — use with care.

Auto-reconnect on Boot

If you ran trust in bluetoothctl, the keyboard reconnects automatically when Linux boots and Bluetooth is on. If it doesn't reconnect, run bluetoothctl connect XX:XX:XX:XX:XX:XX from a wired keyboard or use the GUI Bluetooth settings.

To see paired devices: bluetoothctl paired-devices — this shows your keyboard's MAC address so you don't have to look it up again.
